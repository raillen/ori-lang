## 3. Fix aplicado nesta sessão (strings)

Regra do ADR (`../adr-arc-single-cascade-owner.md`) aplicada aos caminhos
de string do `native_backend.rs`:

1. **Branch `ori_io_print`/`ori_io_eprint`**: libera o +1 de argumentos
   owned (concat, f-string, conversões) **após** o call; borrowed (Var)
   intocado. Era o único caminho de call que não liberava owned args.
2. **`emit_interpolated_string_parts`**: cada concat consome as entradas —
   agora libera o `current` intermediário anterior e o `next` owned
   (conversões escalares alocam managed no runtime: `write_string_parts`
   → `cstring_from_bytes` → `ori_alloc`). Literais/borrowed preservados.

Regressão: `memory_arc.rs` — `print_string_temps_no_leak`,
`fstring_intermediates_no_leak` (19/19 verdes).

## 4. Gap confirmado: maps e sets (próxima sessão)

O mesmo padrão de **bugs pareados que se mascaram** da F1:

- `emit_map_runtime_call`/`emit_set_runtime_call` **não liberam o +1 dos
  args owned** (`maps.set(m, k, make_list(n))` vaza o temp; chave owned
  `"k" + string(1)` vaza em set/get/contains/remove).
- `maps.get` retorna o valor **borrowed** do map, mas `expr_produces_owned_ref`
  trata `Call` como owned → um `const xs = maps.get(...)` **não faz retain**
  e o cleanup do binding **rouba** um +1 que não é dele. Hoje isso "fecha a
  conta" com o temp vazado do set — se corrigir um lado sem o outro,
  vira use-after-free (mesma armadilha da F1).
- Removes estão corretos: `ori_map_remove_raw`/`ori_set_remove_raw`
  desregistram edges (release) no runtime.
- Overwrite de chave está correto no runtime (`ori_map_set_raw` usa
  `ori_arc_update_edge`).

**Fix proposto (3 partes, juntas):** (a) liberar args owned após os calls
de map/set; (b) fazer os paths de `get`/`try_get` **retain** do resultado
managed (aí "Call ⇒ owned" fica verdadeiro); (c) testes red-first:
set/add/get/overwrite/dup com `live_allocations`, incluindo o cenário
"get binding sobrevive à morte do map" (UAF guard).

## 6. Explicitamente NÃO fazer

- Corrigir só o lado do set (leak) sem o lado do get (retain) — UAF.
- Mudar `expr_produces_owned_ref` para tratar todos os intrinsics como
  borrowed — quebraria os calls genuinamente owned; o certo é fazer o get
  cumprir o contrato de owned.
- Tocar no residual de closures (fora do escopo de memória).
